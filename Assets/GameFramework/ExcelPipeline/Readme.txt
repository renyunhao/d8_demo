1.数据表生产线中文件后缀为".xls"、".xlsx"、".xlsm"的文件被认为是程序所需要的，其余的一律不解析
2.以"T"作为前缀的数据表或者文件夹都会将这些表标记为（测试表），测试表寻找相同表名的主表填充其余内容之后导出
3.以"C"作为前缀的数据表或者文件夹都会将这些表标记为（子数据表），子数据表寻找相同表名的主表覆盖主表中内容之后导出
4.每一张数据表中可能存在多个Sheet,以大写字母开头其余字符为字母、下划线、数字命名的Sheet认为是程序所需要的
其余一律不解析(支持一张表多个sheet),同时不需要添加后缀（如：...Data）客户端会再解析时添加统一的后缀（TableData）
5.数据表中的前5行是有特殊意义的不能被占用:第一行表描述
                                       第二行:字段描述
                                       第三行:字段名(以小写字母开头其余字符为字母、下划线、数字命名的Sheet认为是程序所需要的)
                                       第四行:服务端字段类型
                                       第五行:客户端字段类型
6.数据表中的第一列是存在特殊意义的不能被占用： （编号）（id）（int）必须以此格式对数据表的第一列处理

7.字段类型目前支持("int","long","float","double","bool","string","int[]","long[]","float[]","double[]",
"bool[]","string[]","Vector2",,"Vector3","IDCount","IDCount[]","Range","Range[]"),如果要新增类型需要和客户端沟通
8.字段的分隔符暂时只支持( '|'、','、'-' )
9.支持自定义数据类型,但是需要提前沟通加入相应的代码;目前支持自定义数据类型：
Article
{
    "id":50201,"count": 30
}
Range
{
    "min": 10,"max":20
}
